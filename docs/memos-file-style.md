# Memos 文件风格规范

本文档描述了 Memos 功能的文件结构、命名规范、内容格式和 UI 展示风格。

## 📁 文件结构与命名规范

### 文件命名格式

- **格式**: `YYYYMMDD_[标题或ID].md`
- **日期前缀**: 使用 `YYYYMMDD` 格式（如 `20250117`）
- **标题提取**: 如果内容中有 Markdown 标题（`# 标题`），会提取第一个标题作为文件名
- **标题清理**: 只保留字母数字、中文字符、空格和连字符，空格替换为下划线，限制50字符
- **备用方案**: 如果没有标题，使用 8 位随机 nanoid

**示例文件名**:

- `20250117_技术学习笔记.md`
- `20250117_前端开发心得.md`
- `20250117_a1b2c3d4.md` (无标题时使用 nanoid)

### 存储位置

- **默认路径**: `/Memos` （可通过 `WEBDAV_MEMOS_PATH` 环境变量配置）
- **存储方式**: WebDAV 远程存储
- **附件存储**: 在 `assets` 目录，临时文件在 `assets/tmp`

## 📝 文件内容格式

### Frontmatter 结构

```yaml
---
createdAt: "2025-01-17T10:30:00.000Z"   # 创建时间
updatedAt: "2025-01-17T10:30:00.000Z"   # 更新时间  
public: true                            # 公开/私有状态
attachments:                            # 附件信息（可选）
  - filename: "image.jpg"
    path: "/assets/image.jpg"
    contentType: "image/jpeg"
    size: 12345
    isImage: true
tags:                                   # 标签（可选）
  - "技术"
  - "前端/React"
---
```

### 字段说明

- **createdAt**: ISO 8601 格式的创建时间戳
- **updatedAt**: ISO 8601 格式的更新时间戳
- **public**: 布尔值，控制内容可见性（true=公开，false=私有）
- **attachments**: 附件数组，包含文件名、路径、类型、大小等信息
- **tags**: 标签数组，支持层级结构（使用 `/` 分隔）

### 内容特点

- **无标题字段**: Frontmatter 中不存储 title，标题从内容中动态提取
- **Markdown 格式**: 支持完整的 Markdown 语法
- **标签解析**: 自动从内容中提取 `#标签` 格式的标签，支持层级（如 `#前端/React`）
- **附件支持**: 支持图片和文件附件，存储在 frontmatter 中

### 完整示例文件

```markdown
---
createdAt: "2025-01-17T10:30:00.000Z"
updatedAt: "2025-01-17T10:30:00.000Z"
public: true
attachments:
  - filename: "screenshot.png"
    path: "/assets/screenshot.png"
    contentType: "image/png"
    size: 45678
    isImage: true
tags:
  - "技术"
  - "前端/React"
  - "学习笔记"
---

# React Hooks 学习心得

今天学习了 React Hooks 的使用方法，特别是 `useState` 和 `useEffect`。

## 主要收获

- Hook 让函数组件也能使用状态
- `useEffect` 可以处理副作用
- 自定义 Hook 可以复用逻辑

#React #前端开发 #学习笔记

记得明天继续学习 `useContext` 和 `useReducer`！
```

## 🎨 UI 展示风格

### 时间线布局

- **时间线设计**: 垂直时间线布局，带连接线和圆点图标
- **卡片样式**: 白色/深色卡片，圆角阴影，悬停效果
- **紧凑间距**: 最小化卡片内边距，节省空间

### 时间显示

- **友好格式**: 3个月内显示相对时间（如"2小时前"）
- **完整日期**: 超过3个月显示年-月-日格式
- **悬停详情**: 鼠标悬停显示完整时间戳

### 状态指示

- **公开状态**: 蓝色徽章 + 地球图标
- **私有状态**: 琥珀色徽章 + 锁定图标
- **编辑状态**: 显示"已编辑"标识

### 内容渲染

- **简化 Markdown**: 自定义轻量级 Markdown 渲染器
- **标签显示**: 灰色圆角徽章，`#标签` 格式
- **附件网格**: 1:1 比例网格布局，支持点击放大

## ⚡ 功能特性

### 快速编辑

- **管理员编辑器**: 页面顶部快速发布编辑器
- **实时预览**: 支持编辑/预览模式切换
- **拖拽上传**: 支持文件拖拽上传到编辑器

### 权限控制

- **公开/私有**: 支持内容可见性控制
- **管理员功能**: 删除、编辑等管理操作
- **访客模式**: 非管理员只能查看公开内容

### 标签系统

- **自动提取**: 从内容中自动解析 `#标签`
- **层级支持**: 支持 `/` 分隔的多级标签
- **过滤清理**: 自动过滤无效标签格式

### 附件管理

- **多格式支持**: 支持图片、文档等多种文件格式
- **临时上传**: 支持临时文件上传，发布时转为正式附件
- **网格展示**: 附件以网格形式展示，图片支持预览放大

## 🔧 技术实现

### 文件名生成逻辑

1. 获取当前日期，格式化为 `YYYYMMDD`
2. 尝试从内容中提取第一个 Markdown 标题
3. 如果有标题，清理并截取前50字符作为文件名
4. 如果无标题，生成8位随机 nanoid
5. 组合为最终文件名：`${datePrefix}_${titleOrId}.md`

### 标签提取规则

- 使用正则表达式 `/#([a-zA-Z0-9\u4e00-\u9fa5_\-\/]+)/g` 匹配标签
- 支持中文、英文、数字、下划线、连字符和斜杠
- 过滤空标签和无效格式
- 支持层级标签（如 `#前端/React/Hooks`）

### 时间处理

- 创建时设置 `createdAt` 和 `updatedAt` 为当前时间
- 更新时只修改 `updatedAt` 字段
- 前端显示时根据时间差选择友好格式或完整日期

## 📋 设计理念

这种设计体现了**轻量化、时间线式、快速记录**的理念：

- **轻量化**: 最小化 frontmatter 字段，动态提取标题
- **时间线式**: 按时间顺序展示，突出时间流逝感
- **快速记录**: 支持快速编辑、标签自动提取、拖拽上传等便捷功能

适合记录日常想法、灵感、学习笔记和快速思考。
