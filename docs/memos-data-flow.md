# 闪念(Memos)功能完整数据处理流程

## 1. Markdown 文件格式规范

### 1.1 文件存储结构

- **存储位置**: `memos/` 目录下

#### 1.1.1 界面创建闪念时的文件命名规则

- **文件命名格式**: `{datePrefix}_{titleSlug}.md`
- **日期前缀格式**: `YYYYMMDD` (如: `20241201`)
- **标题 Slug 生成**: 从内容提取标题后转换为 URL 友好格式
- **备用方案**: 如果无法生成 titleSlug，使用 nanoid 代替
- **示例**: `20241201_react-learning.md` 或 `20241201_a8b9c2d1.md`

#### 1.1.2 内容源同步时的文件处理

- **兼容性原则**: 接受任何合法的 `.md` 文件名格式
- **现有文件**: 保持原有文件名不变，不强制重命名
- **slug生成**: 根据文件名和内容智能生成数据库slug

### 1.2 Frontmatter 字段定义

| 字段名 | 类型 | 必需 | 默认值 | 用途说明 |
|--------|------|------|--------|----------|
| `title` | string | 否 | 从内容提取 | 闪念标题，从 H1-H7 标题提取，找不到则无标题 |
| `public` | boolean | 否 | `true` | 是否公开显示 |
| `tags` | string[] | 否 | `[]` | 标签数组，支持中英文标签 |
| `publishDate` | string | 是 | 当前时间 | 发布时间，ISO格式，用于排序 |
| `updateDate` | string | 否 | - | 更新时间，ISO格式 |
| `attachments` | object[] | 否 | `[]` | 附件信息数组 |

### 1.3 Slug 生成逻辑

#### 1.3.1 界面创建闪念时的 slug 生成

- **自动生成**: 每个闪念创建时自动生成唯一的 nanoid 作为 slug
- **与文件名分离**: 数据库 slug 与文件名中的 titleSlug 是两个独立的概念
- **唯一性保证**: nanoid 确保全局唯一性，避免冲突

#### 1.3.2 内容源同步时的 slug 生成

- **智能转换**: 根据文件名和内容智能生成 slug
- **转换规则**: 使用拼音等转换方式生成 URL 安全的 slug
- **兼容处理**: 对于各种文件名格式都能生成合适的 slug
- **去重处理**: 如果生成的 slug 与现有记录冲突，自动添加后缀确保唯一性

#### 1.3.3 数据库存储规则

- **存储内容**: 数据库的 slug 字段存储上述两种方式生成的结果
- **URL 路由**: slug 作为 URL 路由的唯一标识符
- **唯一性约束**: 数据库层面确保 slug 字段的唯一性

### 1.4 字段兼容性处理

- **发布时间兼容**: `date` 或 `createdAt` → `publishDate`，`date` 字段优先
- **更新时间兼容**: `updatedAt` → `updateDate`
- **排序规则**: 默认使用 `publishDate` 进行时间排序

### 1.5 正文内容格式要求

- 支持标准 Markdown 语法
- 支持内联标签格式 (如: `#技术 #React`)
- 支持图片相对路径引用
- 支持代码块和数学公式

### 1.6 附件路径语义（持久化 vs 运行时）

为保证内容可迁移性与 FS-only 运行，附件/图片路径遵循两层语义：

- **持久化（Persisted）**：Markdown 与数据库元数据中只允许保存**规范化相对路径**（例如 `./assets/a.png`），不得包含 `/api/files/`，也不得把 `webdav/local` 等源标识写入内容本身。
- **运行时（Runtime）**：渲染与下载时才把相对路径映射为 `/api/files/<source>/<resolvedPath>` 以便浏览器访问；该映射不得回写污染内容文件或数据库。

## 2. 数据处理流程

### 2.1 界面创建闪念流程

1. **用户输入** → 编辑器组件收集内容
2. **标题提取** → 从 H1-H7 标题中提取，找不到则不存在标题
3. **标签提取** → 从正文内容中提取 `#标签` 格式的标签
4. **文件名生成** → 根据日期和标题生成规范文件名 `{datePrefix}_{titleSlug}.md`
5. **数据库 slug 生成** → 自动生成唯一的 nanoid 作为数据库 slug
6. **Frontmatter 构建** → 组装元数据信息
7. **内容源存储** → 写入完整的 Markdown 文件（local 文件系统或 WebDAV；落盘链接为相对路径）
8. **数据库存储** → 保存完整数据到 SQLite

### 2.2 内容源同步数据处理

1. **文件扫描** → 遍历内容源目录中的 .md 文件（local 文件系统或 WebDAV；兼容任何文件名格式）
2. **Frontmatter 解析** → 提取文件头部的 YAML 元数据
3. **内容解析** → 分离正文内容
4. **字段映射** → 处理兼容性字段转换
5. **标签提取** → 从 frontmatter 和正文内容中提取标签
6. **数据库 slug 生成** → 根据文件名和内容智能生成 nanoid slug
7. **摘要生成** → 自动生成内容摘要
8. **哈希计算** → 计算内容哈希用于变更检测
9. **数据库更新** → 同步到 SQLite 数据库

### 2.3 标签处理和内容过滤逻辑

1. **标签来源分离**:
   - **元数据标签**: 从 frontmatter 的 tags 字段获取，用于显示和过滤
   - **正文标签**: 从正文内容中的 `#标签` 格式提取，仅用于内容创建时合并到元数据

2. **显示时的标签处理**:
   - 闪念显示组件只使用元数据中的标签进行展示
   - 正文内容中的 `#标签` 格式需要从显示内容中过滤移除
   - 避免标签在界面上重复显示

3. **标签过滤功能**:
   - 根据用户选择的标签筛选包含该标签的闪念
   - 过滤逻辑基于数据库中存储的元数据标签
   - 支持多标签组合过滤
   - 标签统计显示每个标签对应的闪念数量

4. **内容渲染处理**:
   - 正文 Markdown 转 HTML 时移除内联标签标记
   - 保持正文内容的纯净性，避免标签干扰阅读
   - 标签信息通过专门的标签组件显示

## 3. 技术实现细节

### 3.1 标题提取逻辑

- **提取范围**: 从 H1 到 H7 标题 (`#` 到 `#######`)
- **优先级**: 按标题级别优先，H1 > H2 > ... > H7
- **提取规则**: 匹配行首的标题标记，提取标题文本
- **无标题处理**: 找不到任何标题时，title 字段为空

### 3.2 标签提取逻辑

- **Frontmatter 标签**: 从 YAML 头部的 tags 数组提取
- **内联标签**: 从正文内容中提取 `#标签名` 格式
- **标签合并**: 将两种来源的标签合并去重
- **标签格式**: 支持中英文、数字、连字符的标签名

### 3.3 所见即所得编辑器机制

- **编辑器**: 基于 Milkdown (Crepe) 实现
- **模式切换**: 支持 WYSIWYG、源码、预览三种模式
- **Frontmatter 处理**: 编辑时转换为 YAML 代码块显示
- **实时预览**: 支持边写边看的实时预览功能

### 3.4 图片和附件处理

- **持久化路径**: 统一保存为规范化相对路径（例如 `./assets/<file>`），避免把 `/api/files/...` 写入内容或元数据
- **运行时映射**: 渲染时将相对路径解析并映射为 `/api/files/<source>/<resolvedPath>`
- **附件上传**: 支持拖拽上传图片和文件（上传通过 Files API，落盘仍为相对路径）
- **存储位置**: 附件存储在 memo Markdown 同级的 `./assets/` 目录（例如 `memos/assets/`）
- **灯箱功能**: 图片点击放大查看

## 4. 数据库存储结构

### 4.1 核心字段

- **id**: 文件路径作为唯一标识
- **slug**: URL 友好的标识符
- **title**: 从内容提取的标题
- **body**: 完整的 Markdown 正文内容
- **excerpt**: 自动生成的内容摘要
- **tags**: JSON 格式存储的标签数组
- **publishDate**: 发布时间戳
- **updateDate**: 更新时间戳
- **public**: 公开状态布尔值
- **contentHash**: 内容哈希值用于变更检测

### 4.2 兼容性字段

- **createdAt**: 兼容旧版本的创建时间
- **updatedAt**: 兼容旧版本的更新时间
- **isPublic**: 兼容旧版本的公开状态字段

## 5. 关键特性总结

- **双重存储**: WebDAV 文件存储 + SQLite 数据库存储
- **智能提取**: 自动提取标题(H1-H7)和标签(#格式)
- **灵活编辑**: 支持 WYSIWYG 和源码模式
- **标签分离**: 元数据标签用于显示，正文标签仅用于创建时合并
- **高效过滤**: 支持基于元数据标签的过滤和全文搜索
- **增量同步**: 基于内容哈希的变更检测
- **兼容性**: 向后兼容旧版本的字段格式
- **文件命名**: 基于日期和标题的规范化命名
- **附件支持**: 完整的图片和文件附件功能
