# 管理员仪表盘功能说明

## 概述

管理员仪表盘是一个全新的功能，为网站管理员提供了直观的数据可视化和统计分析界面。通过美观的图表和交互式组件，管理员可以快速了解网站的运营状况和用户活动情况。

## 功能特性

### 1. 总体统计卡片

仪表盘顶部显示四个主要统计卡片：

- **总文章数**: 显示已发布和草稿文章的数量
- **总闪念数**: 显示公开和私有闪念的数量  
- **总评论数**: 显示待审核和已批准评论的数量
- **互动数据**: 显示总反应数和用户数

每个卡片都采用渐变色设计，并具有悬停动画效果。

### 2. 活动日历

活动日历提供两种视图模式：

#### 年视图
- 显示12个月的活动汇总
- 每个月份显示为一个卡片，包含该月的总活动数
- 点击月份卡片可切换到月视图
- 使用热力图颜色表示活动强度

#### 月视图  
- 显示选定月份的日历网格
- 每一天显示为一个方格，颜色深浅表示活动强度
- 悬停显示详细的活动信息（文章、闪念、评论、反应数量）
- 可以返回年视图

### 3. 活动趋势图

趋势图以堆叠柱状图的形式展示指定时间范围内的活动趋势：

- **时间范围**: 支持最近7天、30天、90天
- **数据类型**: 文章、闪念、评论、反应
- **交互功能**: 悬停显示详细数据
- **统计摘要**: 底部显示各类型活动的总数

### 4. 最近活动

显示最近的10个活动，包括：

- **活动类型**: 文章发布、闪念创建、评论提交、反应添加
- **时间信息**: 相对时间和绝对时间
- **视觉标识**: 不同类型使用不同的图标和颜色
- **新活动标记**: 最新的3个活动会有特殊标识

## 技术实现

### 后端API

新增了 `statsRouter` 提供以下API端点：

- `getOverallStats`: 获取总体统计数据
- `getActivityCalendar`: 获取活动日历数据
- `getTrendStats`: 获取趋势统计数据  
- `getRecentActivity`: 获取最近活动数据

### 前端实现

- **框架**: Astro + TypeScript
- **样式**: Tailwind CSS + DaisyUI
- **交互**: 原生JavaScript
- **图表**: 自定义SVG/CSS实现

### 数据库查询

使用Drizzle ORM进行高效的数据库查询：

- 支持时间范围过滤
- 按日期分组统计
- 多表联合查询
- 优化的SQL查询性能

## 使用方法

### 访问仪表盘

1. 以管理员身份登录
2. 访问 `/admin` 管理面板
3. 点击"仪表盘"卡片
4. 或直接访问 `/admin/dashboard`

### 交互操作

- **刷新数据**: 点击右上角的"刷新数据"按钮
- **切换年份**: 使用活动日历的年份选择器
- **查看月份**: 在年视图中点击月份卡片
- **调整趋势**: 使用趋势图的时间范围选择器
- **查看详情**: 悬停在图表元素上查看详细信息

## 样式特性

### 动画效果

- **卡片悬停**: 轻微上移和阴影变化
- **统计卡片**: 渐变光效动画
- **活动单元**: 缩放和高亮效果
- **趋势柱**: 垂直缩放和亮度变化

### 响应式设计

- **移动端**: 单列布局，优化触摸交互
- **平板端**: 两列布局，适中的元素大小
- **桌面端**: 多列布局，完整功能展示

### 颜色系统

- **主色调**: 蓝色（文章）、绿色（闪念）、紫色（评论）、橙色（反应）
- **强度表示**: 5级颜色深浅表示活动强度
- **状态指示**: 不同状态使用对应的语义颜色

## 性能优化

- **数据缓存**: 利用现有的内容缓存系统
- **懒加载**: 按需加载图表数据
- **防抖处理**: 避免频繁的API调用
- **SQL优化**: 使用索引和高效查询

## 扩展性

仪表盘设计具有良好的扩展性：

- **新统计类型**: 易于添加新的统计维度
- **自定义时间范围**: 支持任意时间范围查询
- **新图表类型**: 模块化设计便于添加新图表
- **权限控制**: 基于现有的管理员权限系统

## 注意事项

- 仅管理员可访问仪表盘功能
- 数据实时性取决于缓存刷新频率
- 大量数据可能影响加载性能
- 建议定期清理过期数据以保持性能
