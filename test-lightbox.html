图片灯箱 URL Hash 功能测试

测试说明：

  1. 点击下面的图片，观察浏览器地址栏是否出现 #image-xxxxx 的 hash
  2. 复制带有 hash 的 URL，在新标签页中打开，看是否直接显示灯箱
  3. 在灯箱打开时，使用浏览器的后退按钮，看是否正确关闭灯箱
  4. 按 ESC 键关闭灯箱，观察 URL hash 是否被清除

功能特性：

  • ✅ 点击图片时在 URL 中添加 hash（格式：#image-{imageId}）
  • ✅ 通过 URL 直接打开对应图片的灯箱
  • ✅ 关闭灯箱时清除 URL hash
  • ✅ 支持浏览器后退/前进按钮
  • ✅ 支持键盘 ESC 键关闭

URL 示例：

原始页面：https://example.com/article
打开图片灯箱：https://example.com/article#image-abc123
关闭灯箱后：https://example.com/article

实现的组件：

  • ContentImageHandler.tsx - 处理文章内容中的图片点击
  • ImageLightbox.tsx - 主要的灯箱组件
  • AttachmentGrid.tsx - 处理闪念附件中的图片

技术实现：

  • 使用简单哈希函数为每个图片生成唯一 ID
  • 监听 hashchange 事件处理 URL 变化
  • 监听 popstate 事件处理浏览器导航
  • 使用 window.history.pushState 更新 URL

注意事项：

  • 图片 ID 基于图片路径生成，确保唯一性
  • 支持多种图片路径格式（相对路径、绝对路径、API 路径等）
  • 兼容现有的图片优化和缓存机制
  • 不影响现有的图片点击和缩放功能
